# 文脈クローズ・エンジン設計（Phase 3）

対象: コースD（`ja-3-10k`, `type: 'cloze'`）以降。Phase 4 の較正コース（`calibrate-mine`）もこの提示を再利用する。
根拠: PLAN.md §4.2 / 判断ログ#17（MVP を経ずフル仕様）。

## 中核の設計判断

**cloze は「別カード」ではなく、既存レールカードの"提示モードの切り替え"である。** カード総数は増えない。
ある語の FSRS 安定度がしきい値を超えると、次の復習からフラッシュカード（語→意味の**受容**）でなく、
文中に空欄を作って語そのものを産出させる提示（意味→語の**産出**）へ自動昇格する。
**採点・スケジューリング（recordReview / FSRS）は一切変えない＝純粋に表示層のみ。** これにより
rail エンジン（useStudyBoard のセッション構築・キュー・連打対策・スパークル演出）を丸ごと再利用できる。

## 昇格条件（`src/srs/cloze.ts`）

```
shouldClozePromote(courseType, fsrs) =
  (courseType が 'cloze' | 'calibrate-mine') かつ
  fsrs.state === Review かつ
  fsrs.stability >= CLOZE_PROMOTION_STABILITY_DAYS (=21)
```

- 未習（New）・学習中（Learning/Relearning）は昇格しない＝まず語自体を覚える段階。
- 21日は卒業しきい値（BURN_STABILITY_DAYS=120）の十分手前＝卒業までの長い区間を文脈で鍛える。
- rail(0-3k) コースは常に false（フラッシュカードのまま）。

## データ契約（cloze マークアップ）

`VocabCard.examples[i].cloze?: string` — 学習言語の例文で、見出し語の出現位置を sentinel `｟＿｠` に
置換したもの。日本語コースはデータパイプライン（`build_ja_3_10k.py`）が MeCab トークン境界一致で生成する
（素朴な部分文字列置換だと活用形・複合語の一部を誤って空欄化する。tatoeba-example-substring-bug と同根）。

- 空欄化はトークン境界の一致スパン全体を置換する（活用語尾を含む表層形ごと）。
- クリーンに空欄化できない例文には `cloze` を付けない → そのカードは昇格しても
  `pickClozeExample()` が undefined を返し、**フラッシュカード提示にフォールバック**する（安全な劣化）。
- reveal（めくり後）は空欄でなく `example.text`（原文・完全形）を見せる＝正解の語が自然な形で確認できる。

## 提示（`StudyGrid.tsx` の Tile / `FocusSheet.tsx`）

昇格カードでクローズ可能な例文がある場合のみクローズ提示。それ以外は既存のフラッシュカードのまま。

- **前面（めくり前）**: 「穴埋め」バッジ + 意味（gloss）をヒント + 空欄入りの文。学習者は語を産出。
- **背面（めくり後）**: 見出し語（答え）+ 読み + 原文（完全形）+ 訳。
- 採点意味論は既存と同一: めくらず「知ってる」= easy／めくって「知ってる」= good。
- 見た目の目印: クローズ中のカードは左端に細いアクセント帯（`.tile.cloze`）。

## PLAN §4.2 の他要素の扱い

- **コロケーション導入**: 例文が語を自然な共起（コロケーション）文脈で見せる＝良質な例文データで達成。
  別UI機構は作らない（例文そのものがコロケーション提示）。
- **多義語の語義弁別**: gloss が複数語義を併記する形（JMdict 由来）で当面カバー。
  例文ごとの語義タグ付けは将来のデータ品質改善で対応（別機構は未実装）。
- **未知語タップ→マイニング（読解モード）**: これは §3.5 の単語収穫（マイニング）機能そのもので、
  PLAN §7 では **Phase 6** に割り当てられている。Phase 3 でハーフ実装すると Phase 6 と重複するため、
  **Phase 6 に委ねる**（cloze エンジン本体＝提示＋昇格＋文脈は Phase 3 で完成）。※要 Kohei 確認事項。

## Phase 4（較正）への申し送り

較正コース（`calibrate-mine`）も `isClozeCourse()` に含まれる＝この cloze 提示をそのまま使える。
較正が追加で必要とするのは配置テスト（docs/placement-test-spec.md）と、コロケーション cloze の
「4択」バリアント等（PLAN §4.3）。cloze カードの土台はここで完成しているので、較正は
「昇格した cloze カードに選択肢UIを足す」形で拡張すればよい。
